# 1. Executive Summary

## 1.1 프로젝트 개요

BIDFLOW V2는 CMNTech(첫 번째 고객)을 대상으로 한 **글로벌 입찰 통합 플랫폼**이다.

### 핵심 목표
```yaml
What: 국내+해외 공고 수집 → 저장 → 매칭 → 알림/대시보드 표출
Why: CMNTech 영업팀의 입찰 업무 시간을 3시간 → 35분으로 단축
Who: 초기 고객 CMNTech 영업기획팀 (김영수 과장 중심)
When: 2025년 Q1 파일럿 → Q2 정식 런칭
```

### 성공 정의 (Must Have)
1. **통합 화면**: 국내/해외 공고가 같은 화면에서 리스트업
2. **자동 매칭**: CMNTech 5개 제품별 매칭 결과(점수/이유/추천액션)
3. **글로벌 실동작**: EU TED API + US SAM.gov API 실제 수집
4. **합법성 우선**: 한국 공고는 공식 API 확인 후 확장

## 1.2 Why V2? (V1 대비 개선점)

| 영역 | V1 상태 | V2 목표 |
|------|---------|---------|
| 데이터 소스 | 나라장터 단일 | TED + SAM.gov + 나라장터(합법 확인 후) |
| 멀티테넌트 | 미지원 | tenant_id 기반 고객 분리 |
| 매칭 품질 | 단순 키워드 | 175점 시스템 + Recall 우선 개선 |
| 권한 관리 | 없음 | RLS 기반 데이터 격리 |
| 운영 | 수동 | Health endpoint + 모니터링 훅 |

## 2.2 Pain Points (상세)

### Pain Point 1: 시간 낭비
```
"매일 아침 3시간을 입찰 검색에만 쓰고 있어요.
나라장터만 해도 검색 결과가 수백 개인데,
조달청, 학교장터, 지자체까지 다 봐야 해요.
정작 제안서 쓸 시간이 없어요."
```
- 영향: 일일 3시간+ 낭비
- 정량화: 연간 780시간 (97.5 근무일)

### Pain Point 2: 공고 놓침
```
"팀장님이 '이 공고 왜 놓쳤어?'라고 물을 때마다 식은땀이 나요.
검색어를 다르게 넣거나, 그 사이트를 안 봤거나...
솔직히 전부 다 보는 건 불가능해요."
```
- 영향: 적합 공고 60%+ 놓침
- 정량화: 연간 ₩5억+ 기회 손실 추정

### Pain Point 3: 제안서 반복 작성
```
"비슷한 공고인데 제안서를 매번 처음부터 써요.
템플릿이 있긴 한데, 공고마다 요구사항이 달라서
결국 다 다시 작성하게 돼요. 3시간은 기본이에요."
```
- 영향: 제안서당 3시간+
- 정량화: 월 15건 × 3시간 = 45시간/월

### Pain Point 4: 경쟁사 동향 파악 불가
```
"E+H나 Siemens가 어디에 참여하는지,
어떤 가격으로 따는지 전혀 몰라요.
낙찰 결과 나오면 그제야 알게 되는데,
그때는 이미 늦은 거죠."
```
- 영향: 전략적 대응 불가
- 정량화: 낙찰률 15% (업계 평균 25%)

### Pain Point 5: 해외 입찰 접근 불가
```
"해외 입찰도 하라고 하는데,
TED가 뭔지도 모르고, 영어 공고문을
어떻게 찾아서 봐야 하는지 모르겠어요.
시간도 없고요."
```
- 영향: 글로벌 시장 진출 지연
- 정량화: 해외 매출 0% (잠재 시장 ₩500억+)

## 2.3 Goals (상세)

### Goal 1: 검색 시간 최소화
```yaml
현재: 3시간+/일
목표: 35분/일 (-83%)
측정: 일일 로그인 → 공고 확인 → 종료까지 시간
```

### Goal 2: 공고 놓침 제로
```yaml
현재: 60%+ 놓침
목표: <5% 놓침
측정: (발견 적합 공고) / (전체 적합 공고) × 100
```

### Goal 3: 제안서 작성 효율화
```yaml
현재: 3시간/건
목표: 1시간/건 (-67%)
측정: 공고 선택 → 제안서 완성까지 시간
```

### Goal 4: 낙찰률 향상
```yaml
현재: 15%
목표: 25%+ (+67%)
측정: (낙찰 건수) / (입찰 참여 건수) × 100
```

### Goal 5: 해외 입찰 시작
```yaml
현재: 0건
목표: 월 2건+ 참여
측정: TED/SAM.gov 공고 입찰 참여 건수
```

## 2.4 User Journey Map

### Stage 1: 인지 (Awareness)
```
Trigger: 팀장 지시 "해외 입찰도 알아봐"
Emotion: 불안, 부담
Action: 구글 검색 "글로벌 입찰 플랫폼"
BIDFLOW 접점: 광고/소개 페이지 발견
```

### Stage 2: 탐색 (Consideration)
```
Trigger: BIDFLOW 데모 요청
Emotion: 호기심, 반신반의
Action: 5분 데모 시청
BIDFLOW 접점: 대시보드 첫 화면 경험
```

### Stage 3: 결정 (Decision)
```
Trigger: 실제 CMNTech 제품으로 매칭 시연
Emotion: 놀람, 기대
Action: 파일럿 신청
BIDFLOW 접점: Pro 티어 3개월 무료 제안
```

### Stage 4: 온보딩 (Onboarding)
```
Trigger: 계정 생성, 제품 등록
Emotion: 기대, 약간의 학습 부담
Action: 5개 제품 키워드 설정
BIDFLOW 접점: 가이드 위자드, 첫 알림 수신
```

### Stage 5: 일상 활용 (Usage)
```
Trigger: 매일 아침 알림 수신
Emotion: 편안함, 효율감
Action: 대시보드 확인 → 관심 공고 상세 → 제안서 시작
BIDFLOW 접점: 일일 대시보드, 경쟁사 분석
```

### Stage 6: 확장 (Expansion)
```
Trigger: 3개월 파일럿 성공
Emotion: 신뢰, 확신
Action: Pro 유료 전환, AI 바우처 공동 신청
BIDFLOW 접점: Enterprise 업그레이드, AI 기능 요청
```
