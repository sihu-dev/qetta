## 개요

본 보고서는 유량계(특히 수도미터)에 적용되는 계량법 및 인증 규제를 분석하여, 소프트웨어 업데이트를 통한 계량 파라미터 변경의 법적 위험성과 AI 보조 정보 제공의 합법적 경계선을 명확히 합니다.

### 조사 대상
- **한국**: 계량에 관한 법률, 형식승인 규정
- **국제**: OIML R49 (수도미터), EU MID 2014/32/EU
- **미국**: AWWA C700/C710 표준

---

## 한국 규제 분석

### 1. 법적 프레임워크

#### 계량에 관한 법률
- **목적**: 공정한 상거래 질서 유지 및 산업 선진화
- **적용 범위**: 상거래용 및 증명용 계량 (법정계량)
- **주관 기관**: 국가기술표준원 (KATS)

#### 법정계량기 지정
수도미터는 13종 법정계량기 중 하나로 지정:
- **적용 범위**: 호칭구경 350mm 이하
- **형식승인 필수**: 제조 또는 수입 시 필수
- **검정 대상**: 상거래/증명용 사용 시 필수

### 2. 형식승인 제도

#### 형식승인 요건
```
┌─────────────────────────────────────┐
│     형식승인 (Type Approval)         │
├─────────────────────────────────────┤
│ • 설계 단계에서 기술기준 적합성 검증  │
│ • 측정범위, 성능, 특성 심사           │
│ • 한국기계전기전자시험연구원(KTC) 등   │
│   지정기관이 수행                    │
└─────────────────────────────────────┘
```

**수수료**:
- 기계식 수도미터: 766,410원
- 전자식 수도미터: 2,157,880원

#### 형식승인 변경 요건
> **중요**: 성능에 영향을 주는 제품 변경 발생 시 형식승인 변경 필수

### 3. 검정 및 재검정 제도

#### 검정 제도
- **초기 검정**: 신제품 출하 전 필수
- **검정 수수료**: 구경별 130원~1,710원

#### 재검정 제도
```
사용 기간 만료 → 재검정 또는 교체
         ↓
    정확도 재확인
         ↓
    재사용 가능
```

- **목적**: 경년 변화에 따른 정확도 유지
- **유사 사례**: 가스미터 5년(가정용), 8년(산업용)

### 4. 법적 위반 및 제재

#### 불법 계량기 유형
- 형식승인 미취득 제품
- 검정 미실시 제품
- **성능 변경 후 재승인 미취득 제품** ⚠️

#### 신고 제도
지능형 계량기 유통관리 시스템을 통해 불법·불량 계량기 신고 가능

## 국제 규제 분석 (OIML/EU)

### 1. OIML R49 (국제계량기구 권고)

#### 개요
- **발행 기관**: International Organization of Legal Metrology (OIML)
- **목적**: 법정계량의 국제 조화
- **최신 버전**: OIML R 49-2:2024 (2025년 정식 승인 예정)

#### 적용 범위
- 냉수 음용수 및 온수용 수도미터
- 형식 평가(Type Evaluation) 및 초기 검증(Initial Verification)
- ISO 4064와 완전 조화 (2013/2014년 통합)

#### OIML과 ISO 4064의 통합
```
┌──────────────┐         ┌──────────────┐
│  OIML R49    │ ←─통합─→ │  ISO 4064    │
│  (법정계량)   │  (2013)  │  (산업표준)   │
└──────────────┘         └──────────────┘
         ↓
   단일 글로벌 기술 규정
   (무역 장벽 제거)
```

### 2. EU MID 2014/32/EU (계량기기 지침)

#### 지침 개요
- **발효**: 2014년 2월 26일 채택, 2016년 4월 20일 시행
- **이전 지침**: 2004/22/EC를 대체
- **적용 범위**: EU 전역 계량기 조화

#### 수도미터 관련 규정 (MI-001)
- **Annex III**: 수도미터 기술 요구사항
- **온도 등급 재정의**:
  - T30: 0.1~30°C (냉수미터)
  - T30/90: 30~90°C (온수미터)

#### 핵심 원칙
```
┌─────────────────────────────────────┐
│       New Approach Directive         │
├─────────────────────────────────────┤
│ 1. 필수 요구사항(Essential Req.)     │
│ 2. 적합성 평가 절차                  │
│ 3. CE 마킹 부여                      │
│ 4. EU 전역 자유 유통                 │
└─────────────────────────────────────┘
```

**MID 승인 효과**: 한 회원국에서 승인받은 계량기는 모든 EU 국가에서 사용 가능

#### Article 14.2 규정
> OIML의 표준 문서는 MID 필수 요구사항 준수의 근거로 인정

### 3. 소프트웨어 관련 핵심 규정 (OIML D-31)

#### 법적 관련 파라미터 (Legally Relevant Parameters)

**정의**:
```
┌─────────────────────────────────────┐
│   Legally Relevant Parameters       │
├─────────────────────────────────────┤
│ • 법적 통제 대상 계량기의 파라미터    │
│ • 계량 결과에 영향을 미치는 모든 값   │
└─────────────────────────────────────┘
```

**분류**:

1. **Type-Specific Parameters (형식별 파라미터)**
   - 계량기 형식에만 의존하는 값
   - 형식승인 시 고정됨
   - 법적 관련 소프트웨어의 일부
   - **예**: K-factor, 계량 알고리즘, 유량 보정 곡선

2. **Device-Specific Parameters (개별 기기 파라미터)**
   - 개별 계량기에 의존하는 값
   - 교정(Calibration) 파라미터: span 조정, 보정 계수
   - 설정(Configuration) 파라미터: 최대/최소 용량, 측정 단위

#### 계량학적 중요 소프트웨어 (Metrologically Significant Software)

**정의**:
> 계량학적으로 중요한 기능을 수행하거나 중요 데이터를 포함하는 모든 소프트웨어 모듈

**요구사항**:
```
┌─────────────────────────────────────┐
│   Software Identification           │
├─────────────────────────────────────┤
│ • 버전 번호로 명확히 식별             │
│ • 소프트웨어와 불가분 연결            │
│ • 명령 시 또는 작동 중 표시           │
│ • 인증서에 명시 필수                 │
└─────────────────────────────────────┘
```

#### 소프트웨어 분리 원칙

```
┌──────────────────────────────────────────┐
│             전체 소프트웨어               │
├──────────────────────────────────────────┤
│                                          │
│  ┌────────────────┐  ┌───────────────┐  │
│  │ Legally        │  │ Non-Legally   │  │
│  │ Relevant SW    │  │ Relevant SW   │  │
│  │ (법적 관련)     │  │ (비법적)       │  │
│  │                │  │               │  │
│  │ • 계량 알고리즘 │  │ • UI/UX       │  │
│  │ • K-factor     │  │ • 로깅        │  │
│  │ • 보정 데이터   │  │ • 통신        │  │
│  └────────────────┘  └───────────────┘  │
│                                          │
│  ⚠️ 분리 불가능 시 → 전체가 법적 관련    │
└──────────────────────────────────────────┘
```

#### 소프트웨어 보호 요구사항

**OIML 필수 사항**:

1. **소프트웨어 수정 방지**
   - 기계적 봉인(Mechanical Sealing)
   - 소프트웨어 봉인(Software Seals/Audit Trail)

2. **변경 증거 보존**
   ```
   소프트웨어 수정 발생
         ↓
   이벤트 카운터 증가
         ↓
   초기값과 비교
         ↓
   불일치 시 → 미검증 상태
   (봉인 파손과 동등)
   ```

3. **전자 봉인 방식**
   - 법적 관련 파라미터 변경 시 이벤트 카운터 증가
   - 초기값을 계량기에 영구 표시
   - 현재값과 차이 발생 시 → 검정 무효 상태

#### 소프트웨어 업데이트 요건

**OIML 규정**:
> 소프트웨어가 어떤 방식으로든 수정되면, 새로운 소프트웨어 식별자 필요

**업데이트 유형별 처리**:

| 업데이트 유형 | 법적 관련성 | 요구사항 |
|-------------|-----------|---------|
| **버그 수정 (계량 무관)** | 낮음 | 버전 변경 권장 |
| **UI/UX 개선** | 없음 | 비법적 SW인 경우 자유 |
| **통신 프로토콜 변경** | 없음 | 비법적 SW인 경우 자유 |
| **계량 알고리즘 수정** | 🔴 높음 | **재형식승인 필수** |
| **K-factor 변경** | 🔴 매우 높음 | **재형식승인 필수** |
| **보정 파라미터 변경** | 🔴 높음 | **재검정 필수** |

## 미국 규제 분석 (AWWA)

### 1. AWWA 표준 개요

#### 조직 정보
- **American Water Works Association**: 미국 수도 협회
- **표준 성격**: 산업 권장 표준 (법적 강제성 없음, 단 많은 주정부가 참조)
- **ANSI 승인**: 미국 국가표준(American National Standards)

### 2. 주요 수도미터 표준

#### ANSI/AWWA C700
- **제목**: Cold-Water Meters—Displacement Type, Metal Alloy Main Case
- **적용**: 금속 케이스 양수(Positive Displacement) 미터
- **크기**: 1/2인치(13mm) ~ 2인치(50mm)
- **미터 유형**:
  - Nutating-disc (회전 디스크)
  - Oscillating-piston (진동 피스톤)

#### ANSI/AWWA C710
- **제목**: Cold-Water Meters—Displacement Type, Plastic Main Case
- **적용**: 플라스틱 케이스 양수 미터
- **크기**: 1/2인치(13mm) ~ 1인치(25mm)
- **용도**: 수도 고객 서비스용

#### 기타 관련 표준
- **C708**: Multi-jet meters (다중 제트)
- **C712**: Single-jet meters (단일 제트)
- **C713**: Fluidic oscillator meters (유체 진동)
- **C714**: Magnetic meters (전자식)

### 3. 2024년 업데이트 사항

#### 주요 개정 내용
1. **EPA 납 및 동 규칙(Lead and Copper Rule) 업데이트**
   - 식수 안전성 강화
   - 재료 요구사항 강화

2. **PFAS 규제 추가**
   - Per- and polyfluoroalkyl substances
   - 음용수 내 규제 물질 정보 추가

3. **ASTM 재료 표준 참조 업데이트**
   - 최신 재료 표준 반영

### 4. AWWA 표준의 특징

#### 권장사항 제공
```
┌─────────────────────────────────────┐
│      AWWA 표준이 정의하는 항목       │
├─────────────────────────────────────┤
│ • 최소/정상/최대 유량                │
│ • 시험 절차                          │
│ • 정확도 한계                        │
│ • 재료 요구사항                      │
│ • 작동 특성                          │
└─────────────────────────────────────┘
```

#### AWWA의 역할 제한
> AWWA는 시험을 직접 수행하거나, 제품 인증을 하거나, 실험실 인증을 하지 않음

**의미**:
- 표준 제정 기관일 뿐
- 제3자 시험 기관(UL, NSF 등)이 AWWA 표준 준수 여부 시험
- 많은 주정부가 조달 규격에 AWWA 표준 요구

## 핵심 질문 분석

### 질문 1: SW 업데이트로 Factor 변경이 불법인 이유

#### 법적 근거 (계층별)

**1단계: 한국 계량법 근거**

```
계량에 관한 법률 제14조
         ↓
   형식승인 제도
         ↓
성능 변경 시 재승인 필수
```

**위반 시나리오**:
```
형식승인받은 계량기
    (K-factor = 1.0)
         ↓
SW 업데이트로 K-factor → 1.05 변경
         ↓
❌ 형식승인 변경 없이 성능 변경
         ↓
🚨 불법 계량기 유통
```

**2단계: OIML 국제 기준**

**OIML D-31: General requirements for software controlled measuring instruments**

핵심 조항:
> "Type-specific parameters are fixed at type approval of the instrument."

**해석**:
- K-factor는 Type-Specific Parameter
- 형식승인 시 고정됨
- 변경 시 새로운 형식승인 필요

**3단계: EU MID 근거**

```
┌──────────────────────────────────────┐
│    MID 2014/32/EU 위반 구조          │
├──────────────────────────────────────┤
│                                      │
│  형식승인 계량기 (CE 마킹)            │
│           ↓                          │
│  SW 업데이트로 계량 파라미터 변경      │
│           ↓                          │
│  적합성 평가 무효화                   │
│           ↓                          │
│  CE 마킹 부정 사용                    │
│           ↓                          │
│  시장 유통 금지 대상                  │
│                                      │
└──────────────────────────────────────┘
```

#### 불법인 이유 정리

**1. 법적 측면**

| 항목 | 설명 |
|-----|------|
| **형식승인 우회** | 승인받지 않은 성능으로 작동 |
| **검정 무효화** | 검정 시점과 다른 계량 특성 |
| **소비자 기만** | 인증 표시와 실제 성능 불일치 |
| **상거래 불공정** | 부정확한 계량으로 거래 왜곡 |

**2. 기술적 측면**

```
계량 정확도 = 실제 유량 × K-factor × 기타 보정
                            ↑
                    이것을 바꾸면
                            ↓
                  전혀 다른 계량기
```

**3. 실무적 위험**

- **민사 책임**: 부정확한 계량으로 인한 손해배상
- **형사 책임**: 계량법 위반 처벌
- **행정 제재**: 판매 금지, 회수 명령
- **신뢰 손실**: 제조사 신용도 하락

#### 비유로 이해하기

```
🏷️ 형식승인 = 자동차 안전도 인증

연비 10km/L로 인증받은 차량
         ↓
SW 업데이트로 리미터 해제, 최고속도 증가
         ↓
❌ 안전도 인증 무효
         ↓
🚨 도로 주행 불법
```

계량기도 동일:
- 특정 K-factor로 승인
- SW로 변경 = 다른 계량기
- 재승인 없이 사용 = 불법

### 질문 2: AI "보조 정보"는 합법인 경계선

#### 법적 분석 프레임워크

**핵심 원칙**:
```
┌──────────────────────────────────────┐
│         법적 관련 vs 비법적          │
├──────────────────────────────────────┤
│                                      │
│  법적 계량 (Legally Relevant)        │
│  • 상거래/과금에 사용                │
│  • 검정받은 값                       │
│  • 변경 불가                         │
│                                      │
│  ─────────────────────                │
│                                      │
│  보조 정보 (Auxiliary)               │
│  • 참고용                            │
│  • 법적 효력 없음                    │
│  • 변경 가능                         │
│                                      │
└──────────────────────────────────────┘
```

#### OIML R 76 (저울) 사례

**보조 표시 장치 규정**:
> "Auxiliary indicating devices are permitted only to the right of the decimal sign."

**예시**:
```
공식 계량:  123.45 kg  ← 법정계량 (검정받음)
보조 정보:      .678   ← 추가 정밀도 (참고용)
```

**적용 원칙**:
- 보조 정보는 주 표시보다 **명확히 구분**
- 사용자가 혼동하지 않도록 **시각적 분리**

#### 한국 계량법 사례

**형식승인 제외 계량기**:
```
저울 또는 명판에 다음 표기 시 형식승인 제외:
• 가정용
• 교육용
• 참조용 ← 이것이 핵심
```

**의미**:
- "참조용"으로 명확히 표기하면 법정계량 대상 아님
- 단, 상거래/과금에 사용 금지

### 질문 3: "참고용 표시" vs "공식 계량" 구분

#### 법적 정의

**공식 계량 (Legal Metrology)**

```
┌──────────────────────────────────────┐
│          공식 계량의 요건             │
├──────────────────────────────────────┤
│ 1. 형식승인 취득 계량기               │
│ 2. 검정 합격                         │
│ 3. 검정 유효기간 내                   │
│ 4. 봉인 무결성 유지                   │
│ 5. 법정 용도 사용                     │
│    • 상거래 (물품 판매)               │
│    • 증명용 (공과금, 세금)            │
└──────────────────────────────────────┘
```

**참고용 표시 (Reference/Auxiliary)**

```
┌──────────────────────────────────────┐
│         참고용 표시의 특성            │
├──────────────────────────────────────┤
│ 1. 법적 효력 없음                     │
│ 2. 과금/과세 불가                     │
│ 3. 형식승인 불필요 (한국 사례)        │
│ 4. 검정 불필요                        │
│ 5. 사용 목적                          │
│    • 교육용                           │
│    • 연구용                           │
│    • 모니터링                         │
│    • 의사결정 보조                    │
└──────────────────────────────────────┘
```

#### 구분 기준 (3-Layer Framework)

**Layer 1: 기기 레벨**

| 구분 | 공식 계량 | 참고용 |
|-----|----------|--------|
| **인증** | 형식승인 ✓ | 불필요 |
| **검정** | 필수 ✓ | 불필요 |
| **봉인** | 필수 ✓ | 불필요 |
| **표기** | 검정 증표 | "참고용" 명시 |

**Layer 2: 데이터 레벨**

```
공식 계량 데이터:
├─ 출처: 검정받은 계량기
├─ 처리: 변경 불가 (Read-only)
├─ 저장: 감사 추적 가능
└─ 사용: 법적 문서 근거

참고용 데이터:
├─ 출처: 비검정 센서, AI 추론
├─ 처리: 변경/보정 가능
├─ 저장: 일반 데이터베이스
└─ 사용: 의사결정 참고
```

**Layer 3: 표시 레벨**

**시각적 구분 원칙**:

1. **위계 구분**
```
공식 계량: 큰 폰트, 굵은 글씨, 상단 배치
참고 정보: 작은 폰트, 얇은 글씨, 하단 배치
```

2. **색상 구분**
```
공식 계량: 검은색 (신뢰, 공식성)
참고 정보: 회색, 파란색 (보조성)
경고 정보: 주황/빨강 (주의 환기)
```

3. **아이콘 활용**
```
✅ 공식 계량: 검정 마크, 인증 아이콘
ℹ️  참고 정보: 정보 아이콘
🤖 AI 분석: 로봇/AI 아이콘
⚠️  주의사항: 경고 아이콘
```

## 결론 및 권고사항

### 핵심 발견사항

1. **SW 업데이트로 Factor 변경은 명백한 불법**
   - 한국: 계량법 형식승인 변경 요건 위반
   - 국제: OIML D-31 Type-Specific Parameter 고정 원칙 위반
   - EU: MID 적합성 평가 무효화

2. **AI 보조 정보는 조건부 합법**
   - "참고용" 명시 필수
   - 과금 배제 필수
   - 명확한 시각적 구분 필수
   - 면책 고지 필수

3. **참고용 vs 공식 계량 구분 기준**
   - 형식승인/검정 여부
   - 법적 효력 유무
   - 표시 방법 (시각적 위계)
   - 사용 목적 (과금 vs 참고)
