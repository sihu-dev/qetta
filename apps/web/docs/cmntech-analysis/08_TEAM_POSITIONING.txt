# BIDFLOW 팀 포지셔닝

> **포지션**: 국내 데이터/통계/보안 분석 전문팀

---

## 1. 팀 비전

"데이터 기반 입찰 의사결정의 새로운 표준"

### 핵심 역량

| 영역 | 전문성 | 적용 |
|------|--------|------|
| 데이터 | 공공/산업 데이터 수집/분석 | 입찰 공고 크롤링 |
| 통계 | 낙찰 패턴 분석, 예측 모델 | 확률 기반 의사결정 |
| 보안 | 데이터 보호, API 보안 | 기업 정보 보안 |

---

## 2. 기술 리더십

### 2.1 데이터 엔지니어링

데이터 파이프라인:
[수집] → [처리] → [분석]
나라장터 → 정제/변환 → 통계
TED → 표준화 → 예측
SAM.gov → 저장 → 시각화

### 2.2 통계 분석

- 낙찰 확률 예측: 과거 데이터 기반 모델
- 가격 전략: 경쟁사 패턴 분석
- 시장 트렌드: 시계열 분석

### 2.3 보안 설계

| 레이어 | 구현 | 상태 |
|--------|------|------|
| API 인증 | JWT + API Key | 완료 |
| Rate Limiting | Upstash Redis | 완료 |
| CSRF 보호 | Double Submit | 완료 |
| 데이터 암호화 | AES-256 | 예정 |

## 3. 차별화 전략

### 3.1 국내 시장 집중

- 입찰 실무자 관점 UX
- 한국어 자연어 처리 최적화
- 국내 공공조달 규정 반영

## 8. 팀 원칙

1. **데이터 우선**: 모든 의사결정은 데이터로 검증
2. **보안 기본**: 보안은 기능이 아닌 기본값
3. **사용자 중심**: 실무자가 실제로 쓰는 도구
4. **지속 개선**: 피드백 → 분석 → 개선 사이클
